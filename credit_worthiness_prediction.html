Preprocessing¶

Since we have a lot of categorical variables, we need to preprocess them so that they can be used by the prediction algorithms. The kind of preprocessing depends on the algorithm we use. Tree based models that work well with categorical data generally don't require us to one-hot encode (dummify) the categorical attributes, thus we can just label encode them. Linear models on the other hand are meaningless if categorical attributes with more than one level are included in the training data without one-hot encoding them. They also require continuous attributes to be on the same scale

Model Building¶

The 3 models that we'll build are:

  • Random Forest: Tree based
  • LightGBM (Gradient Boosting): Tree based
  • Logistic Regression: Linear

The 3 models are diverse (bagging, boosting and linear) and depending on the features they have been trained on, they learn differently. Tree based models are great at capturing interactions between multiple categorical attributes. A single decision tree (CART) is prone to overfitting and with the size of our dataset, this is a real issue. Random forests and gradient boosting machines (GBMs) are ensemble based methods that build a number of weak learners and aggregate their output in order to reduce variance, thus they don't face this issue as much as CART.

Once the models are trained, in order to evaluate the performance of the models, this is what we'll do:

  • Calculate ROC AUC or Area Under the Receiver Operating Curve. This is to determine whether the model is 'skilled' enough to differentiate between a positive and a negative case
  • "It is worse to class a customer as good when they are bad (5), than it is to class a customer as bad when they are good (1)" (https://archive.ics.uci.edu/ml/datasets/statlog+(german+credit+data). This will be used as a factor for choosing the best model of the lot. The cost of false positives is 5x higher than the cost of false negatives, thus optimizing the recall on our validation set is important. However the best recall will be achieved when we don't consider anyone as credit_worthy. This will result in poor recall for the positive cases and the result won't be optimal. Since minimizing false positives is important, we will compute the F1 score (harmonic mean of precision and recall) at various thresholds and the threshold where the F1 score is the highest will be considered as the best threshold
  • Once this threshold is computed, we will calculate the number of false positives and false negatives and finally calculate the cost

def evaluate_model(y_test, y_pred, model_name, best_thresh=-1, cost_fp=5, cost_fn=1, test_name='test'):
    # Generate ROC curve values: fpr, tpr, thresholds
    fpr, tpr, thresholds = roc_curve(y_test, y_pred)
    print('The ROC AUC for {} is {} vs the baseline AUC of {}'.format(model_name, 
                                                                      str(auc(fpr, tpr)),
                                                                      str(0.5)))    
    
    # Plot ROC curve
    plt.plot([0,1], [0,1], linestyle='--', label='Chance')
    plt.plot(fpr, tpr, label=model_name)
    plt.xlabel('False Positive Rate')
    plt.ylabel('True Positive Rate')
    plt.legend()
    plt.title('ROC Curve')
    
    # Choosing the best threshold based on F1 score of negative class since misclassifying a 0 as a 1 has a 5x higher cost
    # than vice versa
    if best_thresh == -1:
        list_f1 = []
        range_to_try = np.linspace(0,1,21) 
        for thresh in range_to_try:
            list_f1.append(f1_score(y_test, y_pred>=thresh, pos_label=0))
        best_thresh = range_to_try[np.argmax(list_f1)]
        print('The best threshold is', best_thresh)
    
    # Printing classification report
    print(classification_report(y_test, y_pred>=best_thresh))
    
    # Calculating cost
    conf_matrix = confusion_matrix(y_test, y_pred>=best_thresh)
    num_fp = conf_matrix[0,1]
    num_fn = conf_matrix[1,0]
    cost = num_fp*cost_fp + num_fn*cost_fn
    print('The cost of misclassification on the {} set is {}'.format(test_name, cost))
    
    return(best_thresh)

How do we interpret this graph?

The ROC Curve plots the False Positive Rate (FPR) on the x-axis and the True Positive Rate (TPR) on the y-axis. FPR is the proportion of those cases that we have classified as positive when they are in fact negative. TPR is the proportion of those cases that we have classified as positive when they are in fact positive. The top right corner of the curve has a TPR and FPR of 1 because we have classified everything as positive. Ideally we would want TPR to be as high as possible and FPR to be as low as possible. In this setup, our Area Under Curve (AUC) would be very high. The AUC can also be interpreted as the number of positive cases that are ranked higher (predicted to have a value higher) than the number of negative cases. If our model didn't differentiate between the positive and negative class well, that is, if it randomly chose one to be higher than the other, then the AUC would be 0.5. This would be a chance classifier. The Random Forest model has an AUC of a little over 0.79 which means that the probability that an actual positive case will be predicted to have a value higher than an actual negative case is 0.79

Precision is the proportion of cases that are correctly classified as positive out of those cases that are predicted as positive. Recall is the same as TPR. Ideally our precision and recall should both be as high as possible. In this setup, our F1 score which is the harmonic mean of precision and recall, would also be very high (has a ceiling of 1). With the best threshold we have an F1-score of 0.8 on the positive class. These scores are lesser for the negative or defaulters class. The threshold is set to a 0.65 which means that only those customers that have a high probability of being credit worthy are predicted as credit worthy.

# Dummifying the categorical attributes and scaling the continous ones
def dummify_and_scale(X, categ_cols, cont_cols, scaler=None):
    list_dummy = []
    for col in categ_cols:
        X_out = pd.get_dummies(pd.Categorical(X[col], categories=df_preprocessed[col].unique()),
                               drop_first=True)
        X_out.columns = [col + '_' + str(dummy) for dummy in X_out.columns] # appending level to the dummy column name
        list_dummy.append(X_out)
    X_out = pd.concat(list_dummy, axis=1)
    X_out.index = X.index
    
    if scaler == None:
        scaler = StandardScaler()
        scaler.fit(X[cont_cols])
    X_scaled = pd.DataFrame(scaler.transform(X[cont_cols]), columns=cont_cols, index=X.index)
    
    X_out = pd.concat([X_out, X_scaled], axis=1)
    return(X_out, scaler)

def decile_analysis(pred):
    pred_deciles = pd.qcut(pred, 10, labels=np.arange(0,10))
    dict_deciles = {0:'Bottom Decile', 9:'Top Decile'}
    attributes_checked = []
    counter = 0
    for attr in significant_attributes:
        if attr[:-2] in categ_cols: # dummified columns
            attr = attr[:-2]
        if attr in attributes_checked: # to skip dummy attributes
            continue 
        for decile, descr in dict_deciles.items():
            print(descr)
            if attr in categ_cols:
                attr_values = X_test[pred_deciles==decile][attr].value_counts(normalize=True)
                print('Attribute: {}, Most common level: {}, Likelihood: {}'.format(attr,
                                                                            dict_le[attr].inverse_transform(attr_values.index)[0], 
                                                                            attr_values.iloc[0]))
            else:
                attr_values = X_test[pred_deciles==decile][attr]
                print('Attribute: {}, Mean: {}'.format(attr,
                                                       np.mean(attr_values)))
        attributes_checked.append(attr)
        print('\n')
        counter += 1

        if counter == 5:
            break
